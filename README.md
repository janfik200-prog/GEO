## 📌 О проекте

Разработка метода прогнозирования золото‑урановых рудных узлов средствами
машинного обучения по геолого‑геофизическим данным.

> **Гипотеза.** ML‑подход даёт более качественный прогноз, чем классический
> критериальный анализ (метод ГИС Интегро «Таксономия по критериям»), поскольку
> способен учитывать сложные нелинейные сочетания геологических факторов.

Данные: **ФГБУ «ВНИГНИ»**, Отделение Геоинформатики (ГИС Интегро).

## 🔬 Ключевые результаты

- **На Анабаре (63 метки Au‑U) гипотезу строго доказать нельзя** — меток слишком
  мало. Раннее «ML в разы лучше» оказалось циркулярным: качество мерили на
  собственных обучающих точках. Урок зафиксирован в методике — **качество
  измеряется только на held‑out реальных точках** (пространственная блочная CV,
  `lift@N%`), никогда на псевдометках.
- **Доказательство перенесено на датасеты с тысячами меток** (ветка `part2`):
  | Датасет | Тип | Результат |
  |---------|-----|-----------|
  | **CMMI / Lawley (USGS)** | Zn‑Pb, США + Канада | ML лучше критериального: **ΔAUC +0.08…+0.11** |
  | **CMMI / Lawley (USGS)** | Zn‑Pb, Австралия | **ΔAUC +0.15** (бутстрэп‑ДИ не включает 0) |
  | **NURE + MRDS** | Au / U, США‑Запад | радиометрия NURE поднимает уран **AUC 0.65 → 0.82**; для золота бесполезна |
- **Признаки коммодити‑специфичны:** радиометрия (eU/K/eTh) помогает урану и не
  помогает золоту — фиксированно‑весовой критериальный этого не умеет, ML
  подстраивается под тип оруденения.

## 🧭 Критериальный baseline

Точная реконструкция метода ГИС Интегро **«Таксономия по критериям»**:

```
расстояние до фактора
  → степенная трансформация (симметризация гистограммы)
  → min‑max нормировка каждого критерия в [0, 1]
  → взвешенное L1‑расстояние до эталона‑минимума (мера Плюты)
```

Веса критериев восстановлены регрессией по нативному
`prognoz.prognoz.property` (**R² = 1.000**, Spearman = 1.000). Используется как
честный baseline в валидации, в боевой ML‑прогноз не входит.

## ⚙️ Метод прогноза

1. **Сетка** 500 × 500 м по маске территории `svita_new`.
2. **Признаки:** расстояния до геологических слоёв → близость через убывающую
   экспоненту; взаимодействия факторов (`tect_combo`, `coincidence_score`,
   `tect_only_penalty`); производные рельефа Copernicus DEM (уклон, кривизна,
   TPI, шероховатость).
3. **presence‑background:** positive — реальные рудопроявления, фон — случайная
   выборка ячеек. Псевдометки не используются (вносили циркулярность).
4. **Ансамбль RF + GB** с усреднением по подвыборкам фона → `prospectivity`;
   неопределённость = разброс членов ансамбля.
5. **Золотые зоны:** верхние 2 % площади → связные компактные ядра (8‑связность,
   ≥ 4 ячеек), ранжирование по силе с бонусом за магматизм и совпадение факторов.

## 📊 Валидация

Пространственная блочная кросс‑валидация (`StratifiedGroupKFold` по блокам 10 км),
показатель качества — **lift** попадания held‑out точек в top‑N % площади.

| Метод | lift @ top‑10–15 % |
|-------|--------------------|
| ML (ансамбль RF + GB) | ~ **2.3–2.4×** |
| Критериальный `geo_score` | ~ 1.0–1.3× (≈ случайный) |

> Гипотеза проверяется именно так — на held‑out реальных точках, а **не** через
> ROC‑AUC на псевдометках.

## 💾 Данные и кэш

Входные данные ГИС Интегро — в `data/Gis-integro/` (шейп‑файлы `shp_dbf/` +
нативный расчёт `Расчет/`). При отсутствии распакованной папки пайплайн сам
распаковывает `data/Прогноз.zip`.

Внешние данные (рельеф Copernicus DEM, геофизика GMT) скачиваются идемпотентно в
постоянный кэш `datacache/` (пути — через `cache_paths.py`, переопределяются
переменной окружения `GIS_CACHE`). Кэш не публикуется в git.
