# Проверка метода на мировых датасетах (часть 2)

Вторая часть проекта по прогнозу золото-урановых рудных узлов. Здесь **гипотеза
проекта проверяется на сторонних датасетах с тысячами меток** — там, где, в
отличие от Анабара (63 метки), статистики достаточно для строгого вывода.

**Гипотеза:** ML-подход даёт более качественный прогноз, чем классический
критериальный анализ (метод ГИС Интегро «Таксономия по критериям»), так как
учитывает сложные нелинейные сочетания геологических факторов.

> Это ветка `part2`. Боевой прогноз по Анабарскому щиту (`src/`, данные ГИС
> Интегро, ноутбуки) — в ветке **`main`**: `git checkout main`.

## Что показано

- **CMMI/Lawley (USGS, Zn-Pb, США + Канада + Австралия):** ML значимо лучше
  критериального — ΔAUC +0.08…+0.11 (США+Канада), +0.15 (Австралия);
  бутстрэп-ДИ не включает 0.
- **NURE + MRDS (Au/U, США-Запад):** признаки коммодити-специфичны — радиометрия
  NURE (eU/K/eTh) помогает урану (AUC 0.65→0.82) и бесполезна золоту;
  фиксированно-весовой критериальный это не умеет. ML побеждает для обоих типов
  оруденения.
- **Критериальный baseline** — точная реконструкция метода ГИС Интегро
  «Таксономия по критериям» (степенная трансформация → min-max → взвешенное
  L1-расстояние до эталона-минимума, мера Плюты).

## Структура

```
experiments/       скрипты проверки метода на сторонних датасетах:
  cmmi_terrain.py       ML vs критериальный (CMMI/Lawley, США+Канада)
  cmmi_enriched.py      то же с расширенным набором признаков
  cmmi_transfer.py      перенос модели между территориями
  australia_maps.py     карты прогноза по Австралии
  nure_uranium.py       Au/U-проверка (NURE + MRDS, США)
  make_nure_compare_fig.py  сравнительная фигура по NURE
  common.py             общий код: чтение гридов, геофизика GMT, метрики, baseline
cache_paths.py     пути постоянного кэша скачанных данных (datacache/, env GIS_CACHE)
requirements.txt   зависимости
```

## Запуск

Скрипты запускаются **из корня репозитория как модули** (`python3 -m ...`):

```bash
source .venv/bin/activate            # Windows (Git Bash): source .venv/Scripts/activate
pip install -r requirements.txt

python3 -m experiments.cmmi_terrain   # ML vs критериальный (США + Канада)
python3 -m experiments.australia_maps # перенос модели, карты по Австралии
python3 -m experiments.nure_uranium   # Au/U-проверка (NURE + MRDS)
```

## Данные и кэш

Внешние данные (геофизика GMT, радиометрия NURE, метки MRDS/CMMI) в репозитории
**не хранятся** — они большие (сотни МБ) и скачиваются в постоянный кэш
`datacache/` (не в `/tmp` — он чистится при перезапуске сессии). Часть данных
(глобальная геофизика GMT) подтягивается автоматически при запуске; метки CMMI и
NURE/MRDS кладутся в кэш вручную из открытых источников USGS. Источники указаны в
презентации проекта.

Пути к кэшу — через модуль `cache_paths.py`, переопределяются переменной
окружения `GIS_CACHE`.

Метод переносится на любую территорию; веса конкретной модели — нет (для другого
типа оруденения транслируется метод, а не обученные веса).
