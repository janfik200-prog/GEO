# Прогноз золото-урановых рудных узлов (Анабарский щит, лист R-48-XI, XII)

Задача проекта — разработать метод прогнозирования золото-урановых рудных узлов
с использованием машинного обучения. **Гипотеза:** ML-подход даёт более
качественный прогноз, чем классический критериальный анализ (метод ГИС Интегро
«Таксономия по критериям»), так как способен учитывать сложные нелинейные
сочетания геологических факторов.

Данные: ФГБУ «ВНИГНИ», Отделение Геоинформатики (ГИС Интегро).

---

## Две части проекта

- **Часть 1 (эта, основная) — золото и уран по Анабару.**
  Боевой пайплайн прогноза + исследовательские прогоны по Анабару.
- **Часть 2 — перенос метода на мировые данные** (США + Канада и Австралия).


---

## Структура

```
src/               боевой Python-пакет (логика прогноза, единственный источник истины)
notebook/          forecast_main.ipynb (прогноз) + validation_report.ipynb (валидация)
tests/             pytest: smoke-тест пайплайна
experiments/       исследовательские прогоны по Анабару/золоту (см. ниже)
data/              входные данные ГИС Интегро (шейп-файлы, нативные расчёты)
Постановка/        PDF с постановкой задачи по золоту
docs II presentation/  презентация + обзор методов прогнозирования
cache_paths.py     пути постоянного кэша скачанных данных
requirements.txt   зависимости
```

### Что внутри experiments/ (часть 1)

Прогоны, относящиеся к основному проекту (Анабар, золото, диагностика боевой модели):

```
common.py                  общий код исследовательских прогонов
anabar_geophys.py          Анабар с полной глобальной геофизикой
anabar_pu.py               Анабар при малом числе меток (PU-подход)
anabar_znpb.py             Zn-Pb по переносу на сетку Анабара
anabar_uranium_transfer.py перенос U-модели (США-Запад) на сетку Анабара
forecast_map.py            карта перспективности боевой модели
forecast_oof.py            карта перспективности в held-out (out-of-fold) режиме
gold_compare.py            сравнение «золота» ML vs ГИС Интегро при одном пороге
overfit_check.py           проверка, что боевая модель не запоминает точки
p1p2.py                    валидация на независимых геохимических аномалиях
validation.py              headless-прогон валидации (ML vs критериальный)
collect_metrics.py         сводная таблица метрик по всем методам
dem_prune.py               прунинг DEM-признаков (Copernicus)
feat_prune.py              прунинг обогащённых признаков
halo_seeds.py              устойчивость ореольных меток по сидам
pu_seeds.py                устойчивость прироста PU по сидам
```

## Запуск

```bash
source .venv/bin/activate
pip install -r requirements.txt

# тесты
python3 -m pytest tests/ -q

# боевой прогноз и валидация
jupyter notebook notebook/forecast_main.ipynb
jupyter notebook notebook/validation_report.ipynb
```

## Данные и кэш

Часть входных данных лежит в `data/` (ГИС Интегро). Внешние данные (глобальная
геофизика GMT, радиометрия NURE, метки MRDS/CMMI) скачиваются в кэш при первом
запуске соответствующих скриптов; путь кэша настраивается в `cache_paths.py`
(переменная окружения `GIS_CACHE`).
